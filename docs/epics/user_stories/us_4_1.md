# US-4.1 – Select Target Account and Transfer Funds

## Card

As a **Customer**, I **WANT** to select a target Account and transfer funds so that I can move money between Account without visiting a bank branch.

## Conversation

- The Customer selects a target Account and enters the transfer amount.
- The Bank Backend creates a Transaction with `transactionType = TRANSFER` and `status = PENDING`, debits the source Account, and credits the target Account.
- On success the Transaction is updated to `COMPLETED` and an AuditLog entry is written.
- Corresponds to the happy-path in [Transfer Funds](../../use_case/transfer_funds/transferFunds.md): "Customer selects target Account" → "Customer enters transfer amount" → "Bank Backend creates Transaction" → "Bank Backend updates Transaction (status = COMPLETED)".
- KANO: **Performance** — transfers add convenience; the more seamless, the higher the satisfaction.

## Confirmation

- **Given** the Customer has an active Session and has selected "Transfer"
  **When** the Customer selects a target Account and enters a transfer amount
  **Then** the Bank Backend creates a Transaction (`PENDING`), debits the source Account, credits the target Account, updates the Transaction to `COMPLETED`, and writes an AuditLog entry.
