# US-3.1 – View Account Balance

## Card

As a **Customer**, I **MUST** view my current Account balance on the ATM screen so that I can verify my available funds.

## Conversation

- The Bank Backend creates a Transaction with `transactionType = BALANCE_INQUIRY` and `status = PENDING`, retrieves `Account.balance`, and the ATM displays it on screen.
- The Transaction is updated to `COMPLETED` and an AuditLog entry is written.
- The Account balance is not modified by this operation.
- Corresponds to the full activity diagram in [Check Balance](../../use_case/check_balance/checkBalance.md).
- KANO: **Basic** — balance inquiry is a fundamental ATM function.

## Confirmation

- **Given** the Customer has an active Session and has selected "Balance Inquiry"
  **When** the ATM System requests the balance from the Bank Backend
  **Then** the Bank Backend creates a Transaction (`PENDING`), retrieves the Account balance, the ATM displays it, the Transaction is updated to `COMPLETED`, and an AuditLog entry is written.
