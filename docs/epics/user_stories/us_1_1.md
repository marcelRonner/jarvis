# US-1.1 – Insert Card and Validate

## Card

As a **Customer**, I **MUST** insert my Card into the ATM so that the ATM System can read the Card data and verify the Card is valid and not blocked.

## Conversation

- This is the entry point for every ATM interaction — no other use case can proceed without it.
- The ATM System reads the Card `cardNumber`, `expirationDate`, and `isBlocked` flag.
- If the Card is expired or `isBlocked` is `true`, the ATM retains the Card and displays an error.
- Corresponds to activity diagram steps in [Authenticate](../../use_case/authenticate/authenticate.md): "Customer inserts Card into ATM" → "ATM reads Card data" → "Card valid and not blocked?".
- KANO: **Basic** — the Customer expects this to work; absence causes immediate dissatisfaction.

## Confirmation

- **Given** the ATM status is `ONLINE` and no Session is active
  **When** the Customer inserts a valid, non-blocked Card
  **Then** the ATM reads the Card data and prompts for PIN entry.

- **Given** the ATM status is `ONLINE`
  **When** the Customer inserts an expired or blocked Card
  **Then** the ATM retains the Card and displays an "invalid Card" error.
