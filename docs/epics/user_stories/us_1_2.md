# US-1.2 – Enter PIN and Create Session

## Card

As a **Customer**, I **MUST** enter my PIN so that the Bank Backend can verify my identity and the ATM System creates a Session.

## Conversation

- After successful Card validation the Customer is prompted to enter a PIN via the ATM keypad.
- The Bank Backend validates the PIN against the stored (encrypted) `pin` on the Card record.
- On a valid PIN the ATM creates a Session with status `ACTIVE`.
- Corresponds to activity diagram steps in [Authenticate](../../use_case/authenticate/authenticate.md): "Customer enters PIN" → "Bank Backend validates PIN" → "ATM creates Session (status = ACTIVE)".
- KANO: **Basic** — authentication is a mandatory security prerequisite.

## Confirmation

- **Given** the Card has been validated successfully
  **When** the Customer enters the correct PIN
  **Then** the Bank Backend confirms the PIN, the ATM creates a Session with status `ACTIVE`, and the transaction selection screen is shown.
