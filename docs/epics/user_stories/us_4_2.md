# US-4.2 – Handle Transfer Processing Error

## Card

As a **Customer**, I **NEED** the Transaction to be marked as `FAILED` and Account changes reversed if transfer processing fails so that I do not lose funds.

## Conversation

- If the Bank Backend encounters an error while debiting the source Account or crediting the target Account, the entire operation is rolled back.
- The Transaction status is updated from `PENDING` to `FAILED`.
- An AuditLog entry is written and the ATM displays a "Transfer failed" message.
- Corresponds to activity diagram steps in [Transfer Funds](../../use_case/transfer_funds/transferFunds.md): "Transfer processed successfully?" → (no) → "Bank Backend reverses Account changes" → "Bank Backend updates Transaction (status = FAILED)".
- KANO: **Basic** — fund safety on failure is a non-negotiable expectation.

## Confirmation

- **Given** a Transaction with status `PENDING` exists for a transfer
  **When** the Bank Backend encounters a processing error
  **Then** the Bank Backend reverses all Account changes, updates the Transaction to `FAILED`, writes an AuditLog entry, and the ATM displays "Transfer failed".
