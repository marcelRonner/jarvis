# US-2.5 – Handle Cash Dispensing Failure

## Card

As a **Customer**, I **NEED** the Transaction to be marked as `FAILED` and my Account debit reversed if cash dispensing fails so that I do not lose funds.

## Conversation

- If the CashDispenser encounters a mechanical or other error after the Account has been debited, the system must roll back.
- The Bank Backend reverses the Account debit and updates the Transaction status from `PENDING` to `FAILED`.
- An AuditLog entry is written to record the failure.
- The ATM displays a "Dispensing error" message.
- Corresponds to activity diagram steps in [Withdraw Cash](../../use_case/withdraw_cash/withdrawCash.md): "Cash dispensed successfully?" → (no) → "Bank Backend reverses Account debit" → "Bank Backend updates Transaction (status = FAILED)".
- KANO: **Basic** — data integrity and fund safety are non-negotiable.

## Confirmation

- **Given** a Transaction with status `PENDING` exists and the Account has been debited
  **When** the CashDispenser fails to dispense cash
  **Then** the Bank Backend reverses the Account debit, updates the Transaction to `FAILED`, writes an AuditLog entry, and the ATM displays "Dispensing error".
