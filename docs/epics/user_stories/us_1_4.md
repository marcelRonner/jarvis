# US-1.4 – Block Card After Failed PIN Attempts

## Card

As a **Customer**, I **NEED** my Card to be blocked after the maximum number of failed PIN attempts so that unauthorised access to my Account is prevented.

## Conversation

- When the last remaining PIN attempt fails, the Bank Backend sets `Card.isBlocked` to `true`.
- The ATM retains the physical Card and displays a "Card blocked" message.
- No Session is created.
- The Customer must contact the Bank to unblock the Card.
- Corresponds to activity diagram steps in [Authenticate](../../use_case/authenticate/authenticate.md): "Attempts remaining?" → (no) → "ATM blocks Card" → "ATM retains Card".
- KANO: **Basic** — Card blocking is a regulatory and security expectation.

## Confirmation

- **Given** the Customer has exhausted all PIN attempts
  **When** the last PIN entry is invalid
  **Then** the Bank Backend sets `Card.isBlocked` to `true`, the ATM retains the Card, and a "Card blocked" error is displayed.
